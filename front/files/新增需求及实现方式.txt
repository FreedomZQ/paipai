1、支持 iOS 17.4 及以上版本。“Vision + NaturalLanguage + Translation”的组合方案在图片文字识别，分句子，翻译效果上表现不错，并且在隐私保护和系统整合上优势明显。支持离线使用，支持18 种主流语言。
注意：
1）如果用户从手机上删除了系统自带的“翻译”App，该功能将彻底失效并报错。
2）此外，语言模型是按需首次使用时下载的，且用户可能会去设置中删除语言包，因此务必做好可用性检查。
3）按需下载 (On-Demand)：App安装时不包含模型，首次使用时联网下载。优点是下载包体小，用户可按需管理存储空间
4）这套方案的绝大部分功能都能在搭载 Apple 芯片的 Mac 上实现，并且系统要求更低（macOS 13+ 即可运行），而 iPad 和 iPhone 则需要升级至 iOS/iPadOS 17.4 或更高版本
5）由于 Mac 无摄像头，需引导用户通过“文件选择器”上传图片，无法直接拍照
5）路线图与合规准备
建议按以下步骤推进开发，并提前为 App Store 上架做好准备：

原型验证（POC）：编写 Demo 跑通 Vision 识别 → 文本合并 → NLTokenizer 分句 → Translation 翻译 → AVSpeechSynthesizer 朗读的全流程。

UI/UX 设计：设计扫描、结果展示、播放控制等界面，并实现利用坐标信息高亮原文的交互。

完善与测试：处理错误、优化性能，并在不同设备和 iOS 版本上进行充分测试。

应用上架合规准备：由于应用将完全在设备端处理数据，你需要为 App Store 审核准备以下材料：

隐私清单文件：在 Xcode 中创建 PrivacyInfo.xcprivacy，如实声明 API 收集的数据类型。

App Store Connect 隐私标签：提交时勾选“用户内容”等已使用的数据类型，并说明用于“App 功能”。

隐私政策：在应用内提供完整、可访问的隐私政策，明确说明所有处理均在设备本地完成，不上传服务器。
6）需求落地与技术选型
以下是通过原生框架解决技术难点的具体方式：

📷 拍摄与 OCR 识别：使用 Vision 框架的 VNRecognizeTextRequest。建议将识别等级设置为 accurate 以匹配绘本这种较复杂的排版。Vision 会返回每个文本块的坐标和字符串，可供你在 UI 上高亮显示。

✂️ 智能分句（关键）：利用 Natural Language 框架的 NLTokenizer，将识别出的文本合并后，按 .sentence 为单位进行语义分割，从根源上解决物理换行造成的错误切割。

🌐 翻译（核心）：对每个分句调用 Translation 框架进行离线翻译。Translation API 支持自动检测源语言，并将 source 参数设为 nil 即可，与你的使用场景完美契合。

🔊 语音朗读：使用 AVFoundation 框架的 AVSpeechSynthesizer。你只需指定原文和译文文本及其对应的语言标识符（如 en-US, zh-CN），系统就会调用相应语音引擎进行朗读，完全离线。
7）开发实践与优化建议
体验优化：在图片识别时，可依据 VNRecognizedTextObservation 返回的坐标信息，在 UI 上叠加高亮区域，让家长和孩子清楚识别进度。

性能优化：VNRecognizeTextRequest 比较耗时，应在后台线程执行，避免阻塞 UI。对于 AVSpeechSynthesizer，可在播报前预先创建好 AVSpeechUtterance 对象以减少卡顿。

2、添加孩子使用时长记录和监控记录。
每次和每天的累计使用时长，设定明确的留存期（如30天），到期后自动删除。

3、APP用户登陆或者跳过的时候识别用户用的设备环境例如是MAC还是iPad还是iPhone，以及对应的iOS版本等内容
数据最小化原则，App收集的数据必须与其核心功能直接相关且必要。系统API调用需注意：调用特定API（如 UIDevice.current.model 获取设备型号，或 UIDevice.current.systemVersion 获取系统版本）时，苹果会检测其调用行为。只要你的使用目的正当且符合隐私申报，就不会触发审核拒绝。需要弹窗告知并同意。在隐私政策中增加专门章节，明确列出"设备信息"（如设备型号、操作系统版本）的收集行为，并说明其用途（如"用于保障App的基本功能正常运行、进行设备兼容性适配和问题排查"）。
在用户首次启动App或首次登录时，通过弹窗提示用户阅读并明确同意隐私政策


4、弹窗通知页面
增加弹窗通知功能：用于接收后台下发的各类通知（含内容更新提示、活动通知、服务指引等），无需 APP 版本更新，后台可直接修改通知内容、展示频率及展示规则；通知弹窗仅作信息告知，不强制用户操作，支持后台灵活配置弹窗触发时机（如打开 APP 时触发、定时触发），全程不收集用户任何隐私数据，符合 APP Store 合规要求，可根据需求扩展通知类型（如活动、提醒、更新通知等）。
