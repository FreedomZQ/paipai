# paipaiV2 真机测试部署服务：需要修改 / 替换 / 添加的清单

> 生成时间：2026-04-25 03:08（Asia/Shanghai）  
> 适用目标：开发者把后端部署到一台可被 iPhone/iPad 真机访问的测试服务，并用 Xcode / TestFlight / 真机包完成 Apple 登录、IAP、OCR/TTS、PowerSync、账号删除等真链路验证。  
> 重要原则：这份清单用于“真机测试服务”，不等同于最终生产上线清单。测试环境可以接受 Sandbox IAP，但必须避免把测试开关误带到生产环境。

---

## 0. 当前项目关键约束

| 项目 | 当前值 / 约束 | 说明 |
|---|---|---|
| appCode | `paipai_readingcompanion` | 认证、用户、计费、Remote Config、PowerSync 的一级隔离边界。真机测试不要改成旧值 `reading`。 |
| 内部业务域 | `reading` / `reading_` | Java package、资源目录、表名前缀仍是内部实现名，不是对外 appCode。不要简单全局替换。 |
| 首发系统版本 | iOS/iPadOS `18.0+` | release gate、iOS project、App Store 元数据要一致。 |
| API Base URL | 必须是真机可访问的 `https://...` | iPhone 真机不能使用 `127.0.0.1` / `localhost`。 |
| Apple 登录 / IAP | 必须和 bundleId / appCode 对齐 | `clientId`、App Store `bundleId`、iOS `PRODUCT_BUNDLE_IDENTIFIER` 需要一致。 |
| 本地存储隔离 | 基于 `AppIdentity.appCode` | UserDefaults、Keychain、PowerSync DB 都按 appCode 隔离。 |

APPSTORE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----...-----END PRIVATE KEY-----"

# 6) Apple Sign in key 信息

## 2. 后端需要修改 / 覆盖的内容

### 2.1 必须注入的 Spring 环境变量

测试服务启动时至少需要：

```bash
SPRING_PROFILES_ACTIVE=prod
BACKEND_ENV=prod
BACKEND_OPS_TOKEN="请生成一个足够长的随机 token"

DB_URL="jdbc:postgresql://<db-host>:5432/apphub_test"
DB_USERNAME="apphub"
DB_PASSWORD="请使用强密码"

REDIS_HOST="<redis-host>"
REDIS_PORT="6379"
REDIS_PASSWORD="如有则填写"

# 生产 profile 默认不自动迁移；真机测试若是新库，可先手动跑 Flyway，或临时打开一次。
FLYWAY_ENABLED=false
```

中文说明：

- `SPRING_PROFILES_ACTIVE=prod` 会关闭 Swagger UI / OpenAPI，收紧 actuator 暴露面。
- `BACKEND_OPS_TOKEN` 用于 system 运维 / release gate 等接口，不能为空。
- `FLYWAY_ENABLED=false` 是生产安全默认值；测试新库初始化建议用一次性 Flyway 命令，而不是让应用长期自动迁移。

### 2.2 必须覆盖 Paipai app-definition 的 Apple 登录配置

当前结构位于：

```text
backend/src/main/resources/apps/reading/app-definition.yml
```

不要直接把密钥写死进 git；建议用环境变量或部署平台 secret 覆盖以下键：

```properties
backend.apps.paipai_readingcompanion.app.auth.apple.clientId=${IOS_BUNDLE_ID}
backend.apps.paipai_readingcompanion.app.auth.apple.teamId=${APPLE_TEAM_ID}
backend.apps.paipai_readingcompanion.app.auth.apple.keyId=${APPLE_SIGNIN_KEY_ID}
backend.apps.paipai_readingcompanion.app.auth.apple.privateKey=${APPLE_SIGNIN_PRIVATE_KEY}
backend.apps.paipai_readingcompanion.app.auth.apple.audience=https://appleid.apple.com
backend.apps.paipai_readingcompanion.app.auth.apple.environment=production
backend.apps.paipai_readingcompanion.app.auth.apple.tokenEndpoint=https://appleid.apple.com/auth/token
backend.apps.paipai_readingcompanion.app.auth.apple.jwksUrl=https://appleid.apple.com/auth/keys
backend.apps.paipai_readingcompanion.app.auth.apple.revokeEndpoint=https://appleid.apple.com/auth/revoke
backend.apps.paipai_readingcompanion.app.auth.apple.remoteExchangeEnabled=true
```

检查点：

- `clientId` 必须与 iOS 真机包的 Bundle ID / Apple Sign in 配置一致。
- 如果最终 Bundle ID 从 `com.paipai.readalong.v2` 改为 `com.paipai.readalong`，这里也必须一起改。
- Apple 登录失败时优先检查 `aud`、Team ID、Key ID、nonce、bundleId 是否一致。

### 2.3 必须覆盖 App Store / IAP 配置

```properties
backend.apps.paipai_readingcompanion.app.billing.appstore.bundleId=${IOS_BUNDLE_ID}
backend.apps.paipai_readingcompanion.app.billing.appstore.appAppleId=${APP_APPLE_ID}
backend.apps.paipai_readingcompanion.app.billing.appstore.issuerId=${APPSTORE_ISSUER_ID}
backend.apps.paipai_readingcompanion.app.billing.appstore.keyId=${APPSTORE_KEY_ID}
backend.apps.paipai_readingcompanion.app.billing.appstore.privateKey=${APPSTORE_PRIVATE_KEY}

# 真机 Sandbox / TestFlight 测试环境可临时：
backend.apps.paipai_readingcompanion.app.billing.appstore.environment=sandbox
backend.apps.paipai_readingcompanion.app.billing.appstore.allowSandbox=true

# 正式生产必须：
# backend.apps.paipai_readingcompanion.app.billing.appstore.environment=production
# backend.apps.paipai_readingcompanion.app.billing.appstore.allowSandbox=false
```

检查点：

- App Store Connect 中的商品 ID 必须与后端 productMappings 一致：
  - `com.paipai.readalong.family.yearly`
  - `com.paipai.readalong.family.monthly`
  - `com.paipai.readalong.family.multi_child.lifetime`
- 如果测试阶段临时改商品 ID，必须同步修改：
  - 后端 `app-definition.yml` 的 `productMappings`
  - 数据库 `sys_remote_config` 中 `billing_entitlements` / `reading_plan_catalog`
  - iOS StoreKit 商品读取逻辑所依赖的 plan 配置

### 2.4 必须覆盖 release_ios Remote Config

后端数据库里当前有 `release_ios` namespace。真机测试前至少要保证这些值是真实值：

| namespace | config_key | 测试服务建议值 |
|---|---|---|
| `release_ios` | `bundle_identifier` | `${IOS_BUNDLE_ID}` |
| `release_ios` | `development_team` | `${APPLE_TEAM_ID}` |
| `release_ios` | `marketing_version` | 例如 `1.0.0` |
| `release_ios` | `current_project_version` | 递增整数，例如 `1`、`2` |
| `release_ios` | `paipai_api_base_url` | `${TEST_API_BASE_URL}` |
| `release_ios` | `minimum_ios_version` | `18.0` |
| `release_ios` | `minimum_ipados_version` | `18.0` |

可用 SQL 示例（按实际值替换）：

```sql
UPDATE sys_remote_config
SET config_value_json = jsonb_build_object('value', 'com.paipai.readalong'),
    updated_at = now()
WHERE app_code = 'paipai_readingcompanion'
  AND namespace_code = 'release_ios'
  AND config_key = 'bundle_identifier';

UPDATE sys_remote_config
SET config_value_json = jsonb_build_object('value', 'XXXXXXXXXX'),
    updated_at = now()
WHERE app_code = 'paipai_readingcompanion'
  AND namespace_code = 'release_ios'
  AND config_key = 'development_team';

UPDATE sys_remote_config
SET config_value_json = jsonb_build_object('value', 'https://test-api.example.com'),
    updated_at = now()
WHERE app_code = 'paipai_readingcompanion'
  AND namespace_code = 'release_ios'
  AND config_key = 'paipai_api_base_url';
```

中文说明：

- iOS 发布态脚本会读取这些值渲染 XcodeGen 的 `project.release.yml`。
- 这里不能残留 `__FILL_ME__`、`127.0.0.1`、`localhost`。
- `paipai_api_base_url` 必须 HTTPS，否则真机包预检会失败。

### 2.5 云 OCR / TTS 需要替换的密钥

当前默认 provider 指向阿里百炼 / DashScope 配置。真机测试云 OCR/TTS 时需要：

```bash
DASHSCOPE_API_KEY="真实测试 key"
```

数据库中 `cloud_provider` namespace 的 headers 使用 `Bearer ${API_KEY}` 字面量，应用运行时会按 `ocr.apiKeyEnvName` / `tts.apiKeyEnvName` 找环境变量。

检查项：

- `ocr.apiKeyEnvName = DASHSCOPE_API_KEY`
- `tts.apiKeyEnvName = DASHSCOPE_API_KEY`
- 测试服务容器 / 进程环境里确实存在 `DASHSCOPE_API_KEY`
- 不要把 API key 写进 SQL、Git 或前端包

## 4. iOS 真机包需要替换 / 添加的内容

### 4.1 不要手改 `project.yml` 的占位符，使用 release 渲染脚本

进入：

```bash
cd paipaiV2/ios
```

设置环境变量：

```bash
export RELEASE_IOS_BUNDLE_IDENTIFIER="${IOS_BUNDLE_ID}"
export RELEASE_IOS_DEVELOPMENT_TEAM="${APPLE_TEAM_ID}"
export RELEASE_IOS_MARKETING_VERSION="1.0.0"
export RELEASE_IOS_CURRENT_PROJECT_VERSION="1"
export RELEASE_IOS_PAIPAI_API_BASE_URL="${TEST_API_BASE_URL}"
```

生成 release 项目文件：

```bash
python3 scripts/render_release_project_yml.py \
  --source project.yml \
  --output project.release.yml
```

预检：

```bash
scripts/p0_archive_preflight.sh project.release.yml
```

通过后再生成 Xcode project / archive。

### 4.2 XcodeGen / Archive 建议命令

```bash
xcodegen generate --spec project.release.yml

xcodebuild \
  -scheme PaipaiReadAlongV2 \
  -destination 'generic/platform=iOS' \
  -configuration Release \
  clean archive
```

如果本地需要直接装真机调试，可先用 Debug scheme，但仍必须保证：

- `PAIPAI_API_BASE_URL` 是 HTTPS 测试服务地址
- Bundle ID 与 Apple Developer 配置一致
- Signing Team 正确
- 真机可以访问测试 API 域名

## 5. 后端服务启动后必须验证的接口

假设：

```bash
API="https://test-api.example.com"
OPS_TOKEN="实际 BACKEND_OPS_TOKEN"
```

### 5.1 基础健康检查

```bash
curl -i "$API/actuator/health"
```

预期：HTTP 200，body 包含 `UP`。

### 5.2 公共接口

```bash
curl -s "$API/api/v1/bootstrap/config" | jq .
curl -s "$API/api/v1/plans" | jq .
curl -s "$API/api/v1/legal/docs" | jq .
```

预期：

- bootstrap 返回 paywall / locale / feature 信息
- plans 包含 free / family plan
- legal docs 返回中文隐私政策、服务条款、儿童数据说明

### 5.3 release gate / readiness

```bash
curl -s -H "X-Ops-Token: $OPS_TOKEN" \
  "$API/api/v1/system/release-gate" | jq .

curl -s -H "X-Ops-Token: $OPS_TOKEN" \
  "$API/api/v1/system/apps/paipai_readingcompanion/apple/readiness" | jq .

curl -s -H "X-Ops-Token: $OPS_TOKEN" \
  "$API/api/v1/system/apps/paipai_readingcompanion/apple/ops-gate" | jq .
```

预期：

- release gate 不应再出现 `__FILL_ME__`、空 Apple key、bundleId 不一致、最低系统版本不一致等 blocker
- Apple readiness 至少能指出当前缺失项；真机测试前关键项应全部补齐

## 6. 真机测试流程建议

### 6.1 首次启动 / 公共页面

- [ ] 真机安装后能打开 App
- [ ] 不访问 `localhost` / `127.0.0.1`
- [ ] bootstrap 成功
- [ ] plans 成功
- [ ] legal docs 成功
- [ ] 隐私政策 / 服务条款 / 儿童数据说明可在 App 内打开
- [ ] 未登录时不会看到需要正式账号才能使用的权益状态

### 6.2 Apple 登录

- [ ] Apple 登录弹窗正常
- [ ] `identityToken` 可被后端校验
- [ ] `authorizationCode` exchange 正常
- [ ] 后端返回 appCode 为 `paipai_readingcompanion`
- [ ] Keychain session 保存成功
- [ ] 重启 App 后仍能读取登录态
- [ ] logout 后本地 session 清空

重点排错：

- `aud` 不匹配：检查 iOS Bundle ID 与后端 `clientId`
- `invalid_client`：检查 Team ID / Key ID / private key
- `AUTH_REQUIRED`：检查 session 是否发到了 `Authorization: Bearer ...`

### 6.3 儿童档案 / 学习主流程

- [ ] 新建孩子档案
- [ ] 修改孩子档案
- [ ] 每日任务加载
- [ ] 拍照 / 相册选择可用
- [ ] OCR 本地/云端路径按预期工作
- [ ] TTS 本地/云端路径按预期工作
- [ ] 句卡保存 / 复习事件保存
- [ ] 周报当前页 / 历史页可访问

### 6.4 IAP / 恢复购买

- [ ] Paywall 展示 Apple `displayPrice`，不是只信后端写死价格
- [ ] Sandbox 购买成功
- [ ] 购买后交易 intake 成功
- [ ] entitlement refresh 成功
- [ ] 当前权益变为 premium / family
- [ ] 恢复购买成功
- [ ] 失败/取消购买有明确提示，不误发权益

重点排错：

- 商品加载不到：检查 App Store Connect 商品状态、Bundle ID、Sandbox 账号
- 后端验单失败：检查 App Store Server API issuer/key/private key
- 权益不生效：检查 productMappings 和 `reading_plan_catalog` 中的 `appStoreProductId`

### 6.5 PowerSync / 云同步

- [ ] 未开启云同步时不会上传敏感设备信息
- [ ] 开启云同步后 bootstrap 成功
- [ ] token 成功
- [ ] 本地变更能 upload
- [ ] 同账号重装 / 换设备后能同步核心数据
- [ ] 不同 appCode / 不同账号之间数据不互通

当前 iOS 侧要求：

- `deviceId` 不上传
- `deviceModel`、`appVersion` 等需在隐私同意后才上传
- PowerSync 路由必须是 `/api/v1/powersync/paipai_readingcompanion/...`

### 6.6 账号删除

- [ ] App 内能发起删除请求
- [ ] 邮箱验证码流程可用
- [ ] confirm 后后端删除/标记数据
- [ ] Apple revoke 可用或失败时有可解释提示
- [ ] 本地 session 清空
- [ ] 再打开 App 不应仍显示旧账号权益

## 9. 建议的测试服务最小部署结构

个人开发者低运维建议：

```text
测试 API 域名 + HTTPS
  -> Spring Boot backend
  -> PostgreSQL
  -> Redis
  -> 日志文件 / 平台日志
```

最小安全要求：

- HTTPS 证书有效
- 数据库不暴露公网
- Redis 不暴露公网
- `BACKEND_OPS_TOKEN` 足够长
- App Store / Apple 私钥用 secret 注入
- 后端日志不要打印 token / private key / Apple identity token 全量内容
