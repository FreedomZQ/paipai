## 1. 总结论

当前项目已经达到 **“功能主干基本可联调、后端代码门禁基本收口、iOS 可做静态发布预检”** 的程度，但 **还没有达到可直接提交 App Store 的最终上线标准**。

原因不是主干功能没做，而是 App Store 上线必须依赖以下真实外部条件：

1. Mac / Xcode 真编译与 Archive 尚未执行。
2. 真机 Apple 登录尚未验收。
3. App Store Connect 商品 `com.savingsplanet.app.pro.monthly` 与 Sandbox 购买/恢复购买尚未真实验收。
4. 生产 HTTPS API 域名、隐私政策、服务条款、客服链接需要真实公网可访问。
5. `release-gate` 当前 `codeStatus=ready`，但 `externalStatus=blocked`，剩余阻塞为 Apple Team / Apple 登录密钥 / App Store Server API 密钥 / `release_ios.development_team` 等外部配置。

因此当前状态适合进入：

- Mac/Xcode 真编译
- TestFlight 内测
- Sandbox 购买/恢复购买
- 生产配置填充
- 最终 release-gate 复查

还不建议直接上架提交。

### 5.2 推荐的 V1 免费版边界

免费版不要太弱，否则留存低；也不要太强，否则转化低。建议：

- 免费用户可以完整记录“花费”和“省下的钱”。
- 免费用户可以看基础 Dashboard。
- 免费用户每月 100 条记录上限。
- 免费用户可以看到报告入口和部分摘要，但完整趋势、完整周/月报告需要 Pro。
- 免费用户可以继续使用，不要强制订阅；这对 App Store 审核和低法律风险更安全。

### 5.3 推荐的 V1 付费版卖点

Pro Monthly 应聚焦“持续复盘价值”，不要承诺财务收益：

1. 不限量记录。
2. 完整周报/月报。
3. 消费趋势复盘。
4. 更长历史范围。
5. 导出能力。
6. 后续可优先获得云同步。

文案建议避免：

- “保证省钱”
- “投资建议”
- “理财建议”
- “税务建议”
- “自动帮你赚钱”

推荐文案：

- “帮助你复盘消费习惯”
- “看见省下的钱”
- “用记录形成更稳定的消费反馈”
- “报告仅供个人记账和消费复盘参考”

## 6. 后端离线时 App 主要功能可用性的评估

### 6.1 当前可用

后端离线时，已有本地数据的用户应能继续：

- 查看本地记录列表。
- 新增/编辑/删除本地记录草稿或本地缓存记录。
- 查看最近一次 Dashboard 快照。
- 保留 sync checkpoint，等待未来同步。

### 6.2 当前限制

以下能力离线不能承诺：

- 首次 Apple 登录。
- 购买订阅。
- 恢复购买。
- 后端权益刷新。
- 新设备恢复历史数据。
- 跨设备同步。

### 6.3 首发前建议补强

1. 断网启动提示：明确“当前离线，记录会保存在本机”。
2. 本地待同步标记：新增/编辑/删除记录应有 pending 状态，为未来云同步做准备。
3. 本地冲突策略预留：未来同步时按 `updatedAt` / operation log 合并。
4. 首次使用策略：如果坚持 Apple-only，首次登录仍需要联网；如果要完全离线首次使用，需要重新评估匿名本地账号与 Apple IAP 绑定风险。

建议首发策略：保持 Apple-only，但允许“登录后离线继续使用主要记录功能”。这是审核、安全、售后成本之间最稳的平衡。

## 7. 低运维成本与最低法律风险建议

### 7.1 低运维

推荐：

- 首发只保留一个月度订阅，减少价格/权益/恢复购买复杂度。
- 不上云同步，避免多设备冲突、数据丢失、客服成本。
- 所有转化文案、权益限制、报告开关走 DB remote config。
- App Store Server Notifications 可以先观察/记录，首发不依赖复杂自动化运营。
- 后端 release-gate 保持 fail-closed，避免错误配置上线。

### 7.2 最低法律风险

必须：

- App 内明确“仅用于个人记账和省钱复盘”。
- 报告页明确“不构成财务、投资、税务或法律建议”。
- 账号删除入口清晰。
- 恢复购买入口清晰。
- 订阅取消说明：由用户在 Apple 账户订阅页管理。
- 隐私政策说明记录数据、登录信息、购买凭证、账号删除范围。
- 不承诺“保证省钱”或“产生收益”。

## 8. 高付费转化率与留存建议

### 8.1 转化设计

首发不要一上来强付费。建议三处自然转化：

1. 报告锁定页：用户已有记录后再提示“解锁完整报告”。
2. 会员中心：解释 Pro 能力和恢复购买。
3. Dashboard 趋势入口：展示基础摘要，完整趋势付费。

### 8.2 留存设计

V1 可做但不增加太多运维成本：

1. 空状态引导：让用户 30 秒内完成第一笔记录。
2. 本周/本月复盘提示：不依赖推送也能在 App 内展示。
3. 草稿恢复：已具备。
4. 离线可用提示：降低网络失败导致的流失。
5. 报告生成前置条件：例如“记录 3 笔花费 + 1 笔省钱后报告更准确”。这类文案建议 DB 配置。
