# saving DB 配置化与 P0/P1 落地记录（2026-04-26）

## 本轮目标

围绕首发低运维、低法律风险、App Store 审核顺利、高付费转化和留存，先落地以下配置化内容，并补齐周/月报 P0/P1 的基础复盘能力：

1. 记录分类 catalog
2. 免费/付费权益对比页
3. feature flags
4. onboarding / 空状态 / 留存文案
5. App Review Notes 相关说明
6. 周报/月报中的分类聚合、趋势复盘基础数据

## 数据库层

新增迁移：

```text
src/main/resources/db/migration/V22__saving_config_driven_launch_content.sql
```

新增/更新的 `sys_remote_config` namespace：

| namespace | key | 用途 |
|---|---|---|
| `saving_record_categories` | `expense` / `saving` | 记录分类 catalog，客户端只保存 code，展示名由配置控制 |
| `saving_entitlement_matrix` | `v1` | 免费/Pro 权益对比页 |
| `saving_feature_flags` | `ios_v1` | 首发功能开关、灰度和低运维回滚 |
| `saving_onboarding_copy` | `ios_v1` | 登录/引导文案 |
| `saving_empty_state_copy` | `ios_v1` | 记录与报告空状态文案 |
| `saving_retention_copy` | `ios_v1` | 留存提醒文案，明确不承诺收益 |
| `saving_app_review_notes` | `ios_submission_v1` | App Review Notes 非敏感说明 |

说明：App Review Notes 配置中明确禁止存放密钥、密码、私有证书；生产凭证仍必须通过环境变量或密钥管理注入。

### 新增/扩展接口

控制器：

```text
src/main/java/com/apphub/backend/apps/saving/controller/SavingConfigController.java
```

新增接口：

| API | 说明 |
|---|---|
| `GET /v1/config/categories` | 返回 expense/saving 记录分类 catalog |
| `GET /v1/config/entitlement-matrix` | 返回免费/付费权益对比页配置 |
| `GET /v1/config/feature-flags` | 返回 iOS 首发功能开关 |
| `GET /v1/config/onboarding` | 返回 onboarding、空状态、留存文案 |
| `GET /v1/config/app-review-notes` | 返回 App Review Notes 非敏感配置 |

服务：

```text
src/main/java/com/apphub/backend/apps/saving/service/SavingConfigService.java
```

已补中文说明，强调：

- 配置读取统一使用 `app_code=saving` + namespace 隔离，便于未来多 APP 共用后端。
- 客户端必须保留本地兜底，避免配置服务异常影响离线主流程。
- 留存/报告文案只做事实表达，不承诺收益，不构成财务建议。
