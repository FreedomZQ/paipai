# paipaiV2 P0 / P1 审计总结（2026-04-20）

## 总览
- P0：代码侧已完成收口，backend 相关测试通过；前端因当前环境缺少 Xcode，未完成 iOS 编译/模拟器/真机验证。
- P1：主链路已完成开发与静态审计，backend verify 通过；前端同样受限于无 Xcode 环境。

## P0 审计结论

### P0-1 可编译、可联调真前端
- 已具备真实：
  - BackendClient
  - TranslationService
  - TTSService
  - SecureSessionStore
  - AppStorePurchaseService
  - AppleSignInFlow
- App 启动链路已包含：
  1. 读取本地 session
  2. 拉 bootstrap/config
  3. 校验 auth/me（若本地 session 存在，401 时自动清本地 session）
  4. 未登录时保持未认证态，等待 Apple 登录建立正式会话
  5. 刷新 account/home/plans/legal/parent/review/preferences
- 公告链路已替换 mock，使用后端公告 + 本地展示规则缓存
- 备注：当前环境无法验证“Xcode 真编译通过”

### P0-2 迁移旧版逻辑
- 核心服务与关键模型已迁移/重构到 V2 可用状态
- 旧版逻辑未发现阻断主链路的残留 mock 依赖

### P0-3 unified backend 关键能力
- 已完成：
  - 设备事件上报
  - Apple 登录
  - 删除账号验证码确认
  - 用户偏好与语言记录
  - usage session / child daily usage
  - 公告增强规则字段
- backend verify 通过

### P0-4 release 配置口径
