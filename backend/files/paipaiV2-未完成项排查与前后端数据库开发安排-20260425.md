### P1-1：`encrypted_text` 隐私升级已开始落地

位置：

- `paipaiV2/ios/PaipaiReadAlongV2/Core/Services/LocalTextCryptoService.swift`
- `paipaiV2/ios/PaipaiReadAlongV2/Core/Repositories/ReviewCardRepository.swift`
- `paipaiV2/ios/PaipaiReadAlongV2/Core/Sync/PowerSyncSchema.swift`
- `backend/src/main/resources/db/migration/V19__paipai_p1_privacy_and_weekly_snapshot.sql`

- iOS 新增 `LocalTextCryptoService`，使用 `CryptoKit AES.GCM` 加密句卡正文。
- 256-bit 对称密钥保存在当前 App + appCode 隔离的 Keychain 中，不上传后端。
- `encrypted_text` 新格式：`enc:v1:aesgcm:keychain:<base64-combined-sealed-box>`。
- `text_preview` 改为通用占位 `已保存句卡`，UI 展示时优先本机解密 `encrypted_text`，保持现有展示风格，不新增突兀 UI。
- 云同步 payload 不再上传完整 `source_text` / `translated_text`，后端对新 envelope 只保存密文、通用预览和加密版本。
- 数据库新增：
  - `reading_review_card.content_encryption_version`
  - `reading_review_card.content_key_id`

低运维/低法律风险说明：

- 个人开发者无需维护密钥托管服务。
- 后端不持有本地正文解密密钥，降低儿童阅读内容云端明文留存风险。
- 该方案不是“跨设备端到端密钥同步”方案；换机后如没有本机 Keychain 密钥，只能看到通用预览。后续如要多设备可读，需要设计家长授权的密钥迁移/托管，不应把本地密钥直接上传。

测试：

- 新增/更新后端测试，验证新 envelope 不回填明文 `sourceText`，并写入加密版本/keyId。
- iOS 静态 P0 预检通过；仍需 Mac/Xcode 真编译验证 CryptoKit/Keychain 代码。

## 4. 前端开发安排与逻辑

### 4.1 发布配置流程

负责人：iOS。

逻辑：

1. 从后端 DB `release_ios` 获取：
   - `bundle_identifier`
   - `development_team`
   - `marketing_version`
   - `current_project_version`
   - `paipai_api_base_url`
2. 通过 `scripts/render_release_project_yml.py` 渲染 `project.release.yml`。
3. 跑 `scripts/p0_archive_preflight.sh`，禁止：
   - `__FILL_FROM_DB_`
   - `__FILL_ME__`
   - `127.0.0.1`
   - `localhost`
   - 非 HTTPS API。
4. 生成 Xcode 工程并 Archive。

验收标准：

- Release 构建中 `Info.plist` 可读到生产 API。
- 未配置 API 时 release build 直接 fatal，不 fallback localhost。
- TestFlight 包连接生产/预发 HTTPS 后端。

### 4.2 Apple 登录

逻辑：

1. iOS 调起 Sign in with Apple。
2. 拿到 `identityToken`、`authorizationCode`。
3. 调后端 Apple exchange 接口。
4. 后端验证 identity token、可选远端 code exchange，创建/刷新 session。
5. iOS 保存 session token，后续 API 带认证。

验收标准：

- 首次登录创建账号。
- 同 Apple ID 再登录命中同一用户。
- revoke / 删除账号后再次登录行为符合预期。
- 未配置 Apple 后端凭证时 readiness 明确阻断上线。

### 4.3 StoreKit / 权益

逻辑：

1. iOS 展示后端 `/api/v1/plans` 返回的套餐。
2. StoreKit 购买成功后，把 transaction / productId 传后端 intake/verify。
3. 后端校验签名或回源 App Store Server API。
4. 后端写 `sys_purchase_transaction`，计算 entitlement。
5. iOS 拉 `/api/v1/account/me/state` 和 `/api/v1/subscriptions/status` 刷新 UI。

验收标准：

- 月付/年付映射到 `family_access`，客户端显示 `standard_single_child` 权益。
- 终身多孩子映射到 `family_multi_child`，客户端显示 `family_multi_child_lifetime` 权益。
- 恢复购买能刷新权益。
- 未知 productId 不得错误 fallback 到付费权益。

### 4.4 周报 UI

逻辑：

1. 当前周报：调用 `/api/v1/reports/weekly/current`。
2. 历史周报：若 `historyEnabled=true`，先读 `WeeklyReportLocalCache`。
3. 本地无缓存再调用 `/api/v1/reports/weekly/history`。
4. UI 只展示后端返回的 module：
   - 免费版：基础统计、安全总结、基础建议、分享。
   - 标准版：单孩子详情、复习重点、4 周历史。
   - 家庭版：家庭总览、多孩子摘要、12 周历史。
5. `exportReportEnabled=false` 时不显示导出入口。

验收标准：

- 免费用户看不到高级模块占位和锁定预览。
- 后端断网时，已缓存历史可以展示。
- 切换孩子后 standard 只看当前孩子。
- family 可看家庭聚合。

### 4.5 OCR / 翻译 / 朗读

逻辑：

1. OCR 使用 Vision 本地识别。
2. `NLTokenizer` 做句子/词分割。
3. 使用 boundingBox 做识别文本区域高亮。
4. 翻译仅在 iOS 18+ 且语言方向支持时提示下载语言包/执行。
5. TTS 优先本地；云 TTS/OCR 需按权益和家长设置控制。

验收标准：

- 中英文绘本文字识别可用。
- 低光/倾斜/多行文字有可接受提示。
- iOS 18 翻译不可用时不崩溃，有明确提示。
- 18 种语言需求如果仍保留，必须逐个列入真机验收矩阵。
